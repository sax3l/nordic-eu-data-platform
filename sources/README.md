# Source registry

[`sources.yaml`](sources.yaml) is the machine‑readable index of **every website / dataset with recurring B2B data** the platform harvests. The orchestrator reads it to decide *how* to fetch (from `access` + `waf` + `method`) and *when* to re‑harvest (from `refresh`).

## How to use it

- **Add a source** → append an entry with all fields (see the header block in `sources.yaml`).
- **`refresh`** is the recurrence cadence — the scheduler re‑runs the source on this interval to keep data fresh (B2B data decays ~22‑30%/yr, so cadence is first‑class).
- **`method`** points at a runbook in [`../catalog/methods/`](../catalog/methods/).
- **`tier`** drives compliance handling — see [`../docs/COMPLIANCE.md`](../docs/COMPLIANCE.md).
- **`confidence: uncertain`** means the entry is pending verification by the research pass; verified detail lands in [`../country-data/`](../country-data/).

## Per‑source runbooks

Each source also gets a human runbook under `country-data/<country>.md` and, for non‑trivial sites, a dedicated scraper runbook describing: discovery (sitemap/API), extraction selectors, pagination, WAF behaviour, rate‑limit envelope, expected fields, and the fallback chain. These are generated by the documentation pass (see [`../DOCUMENTATION_PLAN.md`](../DOCUMENTATION_PLAN.md)).
